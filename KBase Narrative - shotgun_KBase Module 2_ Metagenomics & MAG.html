KBase Module 2: Metagenomics & MAGs¶

Author: Elle M. Barnes

RIT Environmental Genomics, BIOL 540/640

Overview¶

Last lab, you learned how to assess, assemble, and annotate isolate genomes generated using shotgun sequencing. However, sometimes it is not possible to isolate microbes to pure cultures in order to generate a reference genome. Thus, scientists have developed a method to sequence environmental DNA in high-throughput in a culture-independent manner. One of these methods is metabarcoding (e.g., 16S rRNA gene sequencing) which surveys all extracted DNA for a single marker region. This is a great method for classifying biodiversity broadly, but it doesn't provided any information about potential function--for that, you need to survey more than a single marker gene. Enter shotgun metagenomics!

What is metagenomics?¶

As the name suggests, shotgun metagenomic sequencing is an untargeted method of DNA sequencing that aims to sequence all microbial DNA extracted from a mixed sample (i.e., not a pure culture). It allows researchers to understand all the (microbial) genetic content in an environment. Right now, metagenomics is used primarily for bacterial and archaeal communities since their genomes are often small allowing you to sequence many of them at high enough coverage in a single sequencing run.

Because it does not rely on culturing, it has become a critical technology used by environmental microbiologists for filling in the microbial tree of life. To do this, they use bioinformatics to not only annotate this large abundance of genes, but to reconstruct draft genomes, known as metagenome-assembled genomes (MAGs), for microbes that have no representative isolates. This technology has blown wide open our understanding of microbial diversity, ecology, and evolution leading to the discovery of new lineages of microbes and complete rearrangement of the tree of life!

We will cover shotgun metagenomics in more detail in lecture. You can also find a great primer by Mike Lee here.

Overview of the process¶

Note from KBase creators: While KBase boasts faster processing and run time on many apps over competitors, please keep in mind that large data sets do take time to analyze. Queue times for metagenomic data sets can appear lengthy during periods of high traffic on the servers. Once through the queue, you should enjoy our faster run times of hours and days for complex algorithms over the months it can take using other avenues.

Description of Apps¶

  • FastQC 2 allows users to check the quality of raw sequence data generated by high throughput sequencing pipelines. The results legends differentiate among normal (green tick), slightly abnormal (orange exclamation), and very unusual (red cross) reads.
  • Trimmomatic 3 performs a variety of useful trimming tasks for paired- or single-end Illumina reads improving the overall quality of the data.
  • Kaiju 4,5 generates fast and sensitive taxonomic classification for metagenomic reads by comparing sequences to databases of known microbial proteins. It also generates an interactive metagenomic visualization chart.
  • metaSPAdes 6 assembles metagenomic reads using the SPAdes assembler.
  • MEGAHIT 7 assembles metagenomic reads using the MEGAHIT assembler.
  • IDBA-UD 8 assembles paired-end reads from single-cell or metagenomic sequencing technologies using the IDBA-UD assembler.
  • Compare Assembled Contig Distributions allows the user to view distributions of contig characteristics for different assembly runs.
  • MaxBin2 Contig Binning 9,10 uses nucleotide composition information, source strain abundance, and phylogentic marker genes to perform binning through an Expectation-Maximization algorithm.
  • Assess Genome Quality with CheckM 11 provides a set of tools for assessing the quality of genomes or metagenomes. It also generates estimates of genome completeness and contamination, and allows the user to filter the bins by these quality scores.
  • Extract Bins as Assemblies from BinnedContigs extracts bins from a BinnedContig dataset as Assembly objects.
  • Annotate Microbial Assembly with RASTtk 12 annotates a bacterial or archaeal assembly using the RAST (Rapid Annotations using Subsystems Technology) pipeline.
  • Build GenomeSet allows the user to group Genomes into a GenomeSet.
  • GTDB-Tk Classify 13 provides a taxonomic placement of the Genomes into the GTDB protein phylogenetic marker derived Species Tree.
  • Insert Set of Genomes into Species Tree 14 constructs a phylogenetic tree combining the GenomeSet provided by the user with a set of closely related genomes from the KBase list of species.
  • Annotate and Distill Assemblies with DRAM 15 identifies and summarizes functional markers in Genomes to assess pathway completeness.

Intitial Taxonomic Assessment¶

Your downstream analysis will generate Species Trees of the organisms present in your sample. Classifying the taxonomy with Kaiju is important because it predicts the microbial composition based on protein similarities from NCBI and other databases rather than genome assembly and annotation. This prediction can be used to compare the Species Trees against.

How does it work?¶

Reads are translated into protein sequences and aligned to reference protein sequences. Best matches to proteins are then taxonomically assigned based on protein’s membership in microbial genomes.

De Novo Assembly¶

Now that you have cleaned the reads, you can move on to assembling the reads into contiguous fragments (contigs) and these contigs into larger regional scaffolds. KBase offers several metagenomic assemblers and a tool for comparing their output (similar to QUAST, which we used for isolate genomes). Run each of the three assembly apps below. All are de Bruijn graph-based assemblers.

Note: You can run each assembler at the same time on the server. They will be submitted as individual jobs. You do not need to wait for one to finish before running the next.

metaSPAdes¶

MetaSPAdes often provides the highest contig accuracy and contiguity resulting in the longest, most accurate contigs with high N50 values, especially in low-complexity samples. However, this accuracy comes at a cost: extremely high memory and CPU consumption making it not ideal for massive soil/water datasets.

MEGAHIT¶

MEGAHIT is specialized for high speed and low memory usage on massive datasets, like soil/water. This is because it relies on succinct de Bruijn graph structure. However, it tends to produce slightly less contiguous assemblies compared to metaSPAdes.

Note: You will see that I include two MEGAHIT assembler cells. Run the same input library twice using different parameters but the same input data. The first run should use "meta-large" as its preset. This is a setting catered towards large and complex assemblies. The second run should use the "meta-sensitive" preset. This parameter generates a more sensitive assembly but runs slower.

IDBA-UD¶

IDBA-UD stands for "Iterative De Bruijn Graph Assembler for UnEven Depths." It was specifically designed to handle metagenomic data with uneven sequencing depths (e.g., high coverage of some organisms and low coverage of others) which can naturally happen due to bias towards sequencing of small genomes or those with low GC content. However, it is generally slower than MEGAHIT.

Bin Contigs¶

Having assembled the contigs, the next step is to cluster/group them into bins, each of which corresponds to a putative organism-specific genome. The resulting bins can then be analyzed as single genomes (MAGs), allowing for taxonomic and metabolic profiling. Binning is the step that allows researchers to reconstruct genomes of microbes when direct cultivation is not possible.

Just as you did with your assembly, you will run each of three binning apps. This ensemble approach is common practice in metagenomic analysis to yield the largest number of high quality bins.

Make sure to view the output of each BinnedContigs data object!

Binning algorithms¶

The following three binning algorithms classify our assembled contigs based on two factors:

  1. Nucleotide composition: Specific signature patterns like GC content or tetranucleotide frequencies vary between species and thus can be used to separate and organize groups of contigs
  2. Coverage/Abundance: Contigs from the same genome should have similar coverage levels across samples, thus we can also categorize contigs based on how frequently they are found

MaxBin2¶

MaxBin2 uses an Expectation-Maximization (EM) algorithm based on marker genes and tetranucleotide frequencies to bin contigs. It is known to produce a high number of bins of high purity but can sometimes yield lower completeness, requiring more post-binning refinement.

MetaBAT2¶

MetaBAT2 uses an adaptive, hierarchical clustering approach, for high speed and efficiency for use on large, multi-sample datasets. It generally requires at least 2.5 kb contigs, making it robust but potentially less effective on very small contigs.

CONCOCT¶

CONCOCT uses Gaussian Mixture Models based on sequence composition and coverage. It performs best when many samples are used. It excels at differentiating closely related species that other tools might merge making it ideal for complex microbial communities.

Annotate Genomes¶

What do these sequences code for?¶

Since you now have your bins in Assembly object form (and collected into an AssemblySet object), we will use "Annotate Multiple Microbial Assemblies with RASTtk" to turn them into annotated Genomes using the Rapid Annotation Subsystem Technology (RAST) pipeline (the "tk" at the end stands for "toolkit" and signifies the version of RAST that allows users to build modular pipelines).

RASTtk identifies features like tRNA/rRNA and protein-encoding genes, then uses the SEED database to predict functional roles and reconstruct metabolic networks. The SEED (yes, they always refer to it with "the" in front) organizes functional roles into gene sets for each metabolic subsystem, which is different than other approaches that simply annotate on a gene-by-gene basis.

Note: If you wish to just do a limited number of annotations, you can run them separately with the Annotate Microbial Assembly App. There is also a RAST pipeline for metagenomes (if you skipped the binning step and did not identify MAGs) called MG-RAST.

Once your bins have all been annotated, the annotation App creates a GenomeSet which will be used as input for the downstream analyses.

HMMER profiling and gene set extraction¶

In additional to profiling functional suites of interest, you can perform HMMER scans of the MAGs to capture explicit gene sets into FeatureSet objects for downstream analysis. These HMMER (pronounced "hammer") scans provide more targeted, gene family model annotation for functional assignments.

While DRAM provides an overview of functional groups, HMMER profiling can "zoom in" to functions of interest for detailed comparisons across genomes/environments.

How does it work? HMMER profiling uses Hidden Markov Models (profile HMMs) to identify, align, and search for homologous protein or nucleotide sequences in your data as compared to functional databases. HMMs are probabilistic models that capture the conservation and variability at each position in a sequence family, including insertion and deletion probabilities. The underlying architecture of HMMs make them more sensitive, effective, and accelerated than BLAST alone.

One example of a profile HMM on KBase is the dbCAN2 collection of hidden markov models (HMMs) which was built from the Carbohydrate Active Enzyme (CAZy) database. You will see that combining DRAM and further HMMER profiling approaches yields additional detailed information about genes found for each gene family (in this case, CAZymes).

Summarize your GenomeSet¶

The last major step of any genome pipeline is to summarize the your MAG/genome quality and completeness!

What are the standard genome characteristics?¶

Standard genome characteristics include (and should sound familiar from above):

  • Total length in base-pairs
  • Number of contigs
  • N50 statistic
  • G+C%
  • Count of protein encoding genes (CDS)
  • tRNA and rRNA counts (useful if needing to determine whether a given MAG meets the MIMAG standard)
    • Completeness and contamination scores are also available

The MIMAG standard¶

The Minimum Information about a Metagenome-Assembled Genome (MIMAG) standard is a set of guidelines developed by the Genomic Standards Consortium (GSC) for reporting the quality, completeness, and contamination levels of bacterial and archaeal genomes assembled from metagenomic data. It ensures transparency and reproducibility, detailing assembly metrics, metadata, and, often, the presence of specific genes like 5S, 16S, and 23S rRNA. The ultimate goal is to create a standardized way for evaluating the massive amounts of genomic information used to understand microbial diversity (with an emphasis on the "uncultivated majority") and align this assessment with previous efforts for cultivated isolate genomes and marker gene surveys.

What is "genome completeness" really?¶

Genome completeness defines how much of an organism's entire DNA sequence is present and correctly assembled. We often verified this by the presence of conserved single-copy genes that should be present in all bacteria/archaea because they are fundamental for life. But, it is important to know that these genes often represent just a small proportion of the total genome. Thus, a genome could be labeled as "complete" and still have accessory genes missing from the assembly.

A further conundrum is that when read libraries used in assembly consist only of short read sequencing technologies in conjunction with the metagenome assemblers described here, the duplication of the ribosomal RNA genes usually precludes assembly of the longer ribosomal RNA genes, meaning it is common that even if a MAG possesses high completeness and low contamination, it can lack the 16S gene required to meet the MIMAG standard. Nonetheless, for analyses based on the protein coding complement of the MAG, it may still prove useful.

Classify Viral Clusters¶

You can use vConTACT2 to classify your assemblies in approximately genus-level viral clusters, referred to as "VCs".

A Viral Cluster is a unit of classification. vConTACT2 uses two terms - with subtle differences - in order to classify sequences:

  • A VC is a first-pass classification of viral genomes. This frequently represents a group of genomes within the same genus.
  • A VC Subcluster is a second-pass classification. It uses a pre-calculated distance calculation to refine the VCs. These are high-confidence, genus-level groupings. It is common for there to be no change between VC and VC Subcluster. The final value ("0") represents the subcluster within the original VC. If all members of a VC Subcluster have VC_xx_0, then there was no change. However, if there were further refinements, then the VC Subcluster would be: VC_xx_0, VC_xx_1, VC_xx_2...

vConTACT2 works by using a gene-sharing network to associate viral genomes. The more genes that are shared between two genomes, the higher the probability of those two genomes being phylogenetically related.

Making sense of the results...¶

STEP 1: After you've run vConTACT2, you'll get a table with ALL of the genomes in the analysis. The table can be a bit unwieldly as it contains a lot of rows and columns. The easiest way to manage this (in KBase) is to use the filter function to find YOUR viral genomes. For example, all of your genomes might contain NODE - it's a byproduct of the metaSPAdes assembler. By adding NODE in the appropriate filtering row under the column "Genome" will filter out all the reference genomes. This is not a perfect method--KBase developers are working on this.

STEP 2: Now, how many of our viral genomes are clustered? Add Clustered to the "VC Status" column.

STEP 3: To find out if your viral genomes are associated with any references, requires some data wrangling. One "workaround" in KBase is to sort the table by "VC" and scroll through, keeping track of the "Size" and count of the VC (i.e., how many of each VC id appears in the table). If the "Size" of the VC is greater than the counts of the VC, make note of that VC!

STEP 4: Remove that NODE filter, to reveal the reference identities that also clustered with your genomes (same VC).
